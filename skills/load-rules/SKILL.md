# Load Rules

强制加载项目全部 RULES。这个 skill 比 `docs/agents/domain.md` 的“1% 相关也要读”更严格：不判断相关性，不跳过弱相关规则，只要是仓库中的 RULES 文件就读取。读取后自动识别并记住对当前会话有影响的规则，作为本会话后续工作的硬约束。

## 流程

### 1. 定位规则目录

- RULES 统一位于仓库根 `docs/rules/`，不按 Context 拆分。
- `docs/CONTEXT.md`、`docs/CONTEXT-MAP.md` 是否存在，不影响 RULES 的读取位置。

### 2. 读取规则文件

- 读取根 `docs/rules/` 下所有 Markdown 规则文件。
- 不按文件名、任务范围、Context 相关性过滤；只排除 `.git`、依赖目录、构建产物和明显不是项目文档的目录。
- 如果没有任何 RULES 文件，明确说明“未发现 RULES”后继续当前任务。

推荐命令：

```bash
rg --files -g 'docs/rules/*.md'
```

### 3. 汇总已加载规则

读取后给出一个简短清单：

- 规则短号（可从文件名或路径推导）
- 标题
- 文件路径
- 一句话约束摘要

不要只列路径就结束；必须实际读取文件内容。除非用户要求，不要在回复中完整复述所有规则原文。

### 4. 记住会话相关规则

从已读取的全部 RULES 中筛出对当前会话有影响的规则，形成“会话 RULES 集”：

- 影响当前需求、代码、命名、单位、错误处理、权限、数据流、模块边界、集成方式、测试、验收或提交的规则，都加入会话 RULES 集。
- 只要不确定是否会影响当前会话，就加入会话 RULES 集。
- 在后续回复、计划、实现、验证和复盘中自动遵守会话 RULES 集，不要求用户重复提醒。
- 如果会话目标变化，重新从已加载的全部 RULES 中更新会话 RULES 集；必要时重新扫描新增规则。

### 5. 后续执行约束

- 当前会话后续任务默认遵守已加载的全部 RULES。
- 对会话 RULES 集中的规则，后续每次做决策、写任务、实现代码或验证结果时都要主动套用。
- 任务实现、方案、测试、验收与任一 RULES 冲突时，先按 RULES 修正。
- 确需偏离时，不得静默违反；必须标出冲突规则、说明原因，并交用户裁决。
- 若启动子 Agent，prompt 必须包含已加载 RULES 的短号、标题、路径和原文摘要，不能只传路径。

## 使用场景

- 用户明确担心 RULES 被忽略。
- 任务影响范围大，按需读取的误判成本高。
- 多 Context 仓库中需要跨域一致性。
- review 或 retro 前需要用全部规则做基准。
