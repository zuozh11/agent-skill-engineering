# 约定式提交（Conventional Commits）

**只做 commit，不做 push。** 提交完成后不要自动推送到远程仓库，由用户自行决定何时 push。

## 提交消息格式

```
<类型>[可选 范围]: <描述>

[可选 正文]

[可选 脚注]
```

## 类型说明

| 类型 | 使用场景 |
|------|---------|
| `feat` | 新增功能 |
| `fix` | 修复 bug |
| `docs` | 仅文档变更 |
| `style` | 格式调整（空格、分号等，不影响逻辑） |
| `refactor` | 重构代码（既不修 bug 也不加功能） |
| `perf` | 性能优化 |
| `test` | 添加或修改测试 |
| `build` | 构建系统或外部依赖变更 |
| `ci` | CI 配置变更 |
| `chore` | 其他不涉及 src 或 test 的变更 |
| `revert` | 回退之前的提交 |

## 破坏性变更

两种标记方式：

1. 在类型/范围后加 `!`：`feat(api)!: 移除已废弃的接口`
2. 在脚注中添加 `BREAKING CHANGE:`：
```
feat: 迁移到新的认证系统

BREAKING CHANGE: JWT token 格式变更，所有客户端必须重新认证
```

两者可以同时使用。脚注中的 `BREAKING CHANGE` 必须大写。

## 工作流程

当用户想要提交时：

1. 执行 `git status --short`、`git diff --cached` 和 `git diff` 查看暂存区与工作区变更。
   默认直接暂存并提交当前会话相关的改动，不要因为暂存区为空就先询问用户。
   “当前会话相关”包括：本轮对话中你创建或修改的文件、用户明确指定的文件、
   以及能够从本轮任务上下文清楚判断属于同一个逻辑变更的文件。
2. 如果暂存区已有变更，先判断它们是否属于当前会话或用户明确要求的提交范围：
   - 属于同一范围：保留暂存内容，并把当前会话相关的未暂存改动一并暂存后提交。
   - 不属于同一范围或判断不清：不要擅自混入提交，使用 `request_user_input` 询问提交范围。
3. 如果存在未暂存或未跟踪文件，默认只 `git add` 当前会话相关的文件。
   只有在无法可靠判断哪些文件属于当前会话、同一路径存在用户未要求的混杂改动、
   或可能把敏感/本地配置/大批生成物误提交时，才询问用户。
4. 分析变更内容：
   - 哪些文件发生了变更，为什么
   - 属于哪种类型（功能、修复、重构等）
   - 影响的范围（模块/组件）
   - 是否包含破坏性变更
5. 按以下规则生成提交消息：
   - **标题行**：`<类型>(<范围>): <描述>` — 祈使语气，小写开头，不加句号，不超过 72 字符
   - **正文**（如需解释变更）：说明变更了什么以及为什么，每行不超过 72 字符
   - **脚注**：如有破坏性变更写 `BREAKING CHANGE:`，关联问题写 `Refs: #123`，关联提交写 `Refs: abc1234`（不带 #），协作者信息等
6. 使用生成的消息执行 `git commit`。

## 编写规则

- 标题行不超过 72 字符
- 描述使用祈使语气（"添加"而非"添加了"，"修复"而非"修复了"）
- 描述小写开头（英文时）
- 标题行末尾不加句号
- 标题、正文、脚注之间用空行分隔
- 正文解释"做了什么"和"为什么"，而非"怎么做的"
- 一个提交对应一个逻辑变更 — 如果变更跨越多个关注点，建议拆分

## 范围选择

范围应反映受影响的模块、组件或领域，从以下维度选取：
- 目录名：`feat(auth):`、`fix(api):`
- 组件名：`style(button):`、`test(user-service):`
- 领域概念：`feat(计费):`、`fix(国际化):`、`feat(订单):`、`fix(权限):`

当变更范围较广或无法归入单一模块时，省略范围。

## 示例

简单 bug 修复：
```
fix(订单): 处理空订单号时不再崩溃
```

带正文的新功能：
```
feat(物料): 添加模糊搜索支持

使用 Levenshtein 距离进行近似物料名称匹配。
可通过 `search.fuzzyThreshold` 选项配置阈值。
```

破坏性变更：
```
feat(合同)!: 审批接口返回值使用信封格式包装

BREAKING CHANGE: 合同审批 API 响应现在包装在 `{ data, meta }` 对象中，
不再直接返回数组。

Refs: #456
```

文档变更：
```
docs: 为新开发者添加贡献指南
```

回退提交：
```
revert: feat(物料): 添加模糊搜索支持

Refs: abc1234
```

多脚注：
```
fix(库存): 防止并发扣减时的竞态条件

引入请求 ID 追踪机制，丢弃过期响应。

Reviewed-by: Alice
Refs: #123
```
