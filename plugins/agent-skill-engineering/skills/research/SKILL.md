# Research

启动一个后台 Agent 执行调研，使主 Agent 可以在它阅读资料时继续处理其他工作。

后台 Agent 的任务：

1. 围绕用户的问题查阅一手来源，包括官方文档、源码、标准规范和第一方 API。二手资料只能用于寻找一手来源；每条关键结论都必须追溯到拥有该事实的来源。
2. 将调研结论写入一个 Markdown 文件，并为每条关键结论标注来源。无法通过一手来源确认的内容明确写为“不知道”。
3. 优先沿用仓库已有的调研或笔记目录与命名约定；没有现成约定时选择合理位置，并在完成后说明文件路径。

后台 Agent 除目标调研文件外不得修改其他文件。宿主不支持后台 Agent 时，由主 Agent 按相同要求完成调研。
